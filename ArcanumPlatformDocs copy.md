# ТЗ: Фрактальные 9 сфер жизни (операционная модель)

## 0) Онтология (итоговая, объединённая)

Матрица **3×3**:
**Уровни (горизонталь):** ЯДРО → ПРОЦЕСС → СИСТЕМА
**Валюты (вертикаль):** ЭНЕРГИЯ → ИНФОРМАЦИЯ → МАТЕРИЯ

### 9 сфер

1. **S1 Vitality — Тело/Реактор** *(Ядро×Энергия)*
   Энергия, гормональный фон, выносливость, сон, питание, **систематическая нагрузка/периодизация**, холод/жара.

2. **S2 Mind/Code — Разум/Код** *(Ядро×Информация)*
   Ценности, убеждения, идентичность, метанавыки: внимание, память, обучение, креатив, языки.

3. **S3 Habitat — Среда/Кокон** *(Ядро×Материя)*
   Локация, сцены потока (рабочее место/свет/звук), предметная среда, минимализм, логистика быта.

4. **S4 Action/Vector — Действие/Вектор** *(Процесс×Энергия)*
   Тренировки, производящая рутина, Deep Work, дисциплина «неприятное — первым».

5. **S5 Communication/Influence — Связи/Коммуникация/Влияние** *(Процесс×Информация)*
   Отношения, переговоры, продажи, публичность, бренд, контент-система, комьюнити.

6. **S6 Craft/Production — Производство/Крафт** *(Процесс×Материя)*
   Код/продукт/артефакты, поставка ценности, качество, процесс материализации идей.

7. **S7 Status/Discipline — Статус/Дисциплина** *(Система×Энергия)*
   Накопленная сила воли, репутация, агентность, авторитет.

8. **S8 Network/Library — Сеть/Библиотека** *(Система×Информация)*
   Сеть людей, знаниевая база, цифровые активы, документация.

9. **S9 Capital/Resources — Ресурсы/Капитал** *(Система×Материя)*
   Финансы, инструменты, инфраструктура, активы.

### Над-слои (мета)

* **R0 Orchestrator — Resonance/Manager.** Управление приоритетами, циклами (day/week/month), автоматизация, подбор «домино-шагов».
* **F(x) Purpose/Values — цель-функция смысла.** Скаляр 0–1, умножается на приоритет задач.

### Фрактальность (вшитый резонанс)

Правило **X-in-Y**: внутри каждой сферы Y хранить 8 подпунктов «Sx-в-Y», чтобы каждая сфера содержала отражения остальных.
Пример: «S2-в-S1 = обучение технике движения», «S5-в-S3 = среда, усиливающая коммуникацию» и т. п.

  * Симметричность обеспечивать парными строками.

# 2) Скоринг «домино-действий» (алгоритм ИИ)

**Формула приоритета:**

$$
\text{Score}(a) = \frac{\Big(\sum_{s} w_s \cdot \Delta S_s(a)\Big)\cdot \big(1+\lambda\cdot C(a)\big)\cdot F(a)}{\text{Effort}(a)}
$$

* $\Delta S_s(a) \in [0,1]$ — ожидаемый прирост по сфере s (берём из `expected_effect`).
* $w_s$ — вес сферы из `resonance_weights` (primary↔secondary учитываем; можно суммировать по всем Sy).
* $C(a)$ — количество **уникальных сфер в задаче** (primary + secondary, минимум 1; домино-бонус срабатывает при ≥3).
* $\lambda$ — домино-коэффициент (по умолчанию **0.2**).
* $F(a) \in [0,1]$ — **Purpose-согласованность** (`purpose_score`).
* `Effort(a)` — трудозатраты (0.5..5).

**Планировщик:**

* **Daily 00:00 (Europe/Zurich):**

  1. Пересчитать `task_scores`.
  2. Выбрать **1–3 задачи** с максимальным Score → state=`planned`→`doing`.
  3. Записать `events(type='DAILY_PLAN')`.
* **Weekly:**

  1. Предложить 1 большой «рычаг» (Score высокий + затрагивает ≥4 сфер).
  2. Предложить «чистку трения» — одну низкоскоринговую задачу к удалению/делегированию.
* **Monthly:**

  1. Пересчитать матрицу весов на основе фактических приростов KPI (EM-подобная корректировка).

# 4) Агентная архитектура

## 4.1 Оркестратор (R0)

**Системный промт (фрагмент):**

> Ты — Orchestrator. Твоя цель — максимизировать суммарный межсферный резонанс. Используй матрицу весов, формулу Score и цель-функцию Purpose. Раз в день выбери 1–3 домино-задачи с наибольшим сквозным эффектом на ≥3 сферы. Если данных не хватает — запроси у профильных агентов соответствующие KPI/контекст.

**Функции:**

* `collect_context()` — тянет KPI, незавершённые задачи.
* `score_tasks()` — пересчёт Score.
* `propose_domino_plan()` — выдаёт список задач и обоснование (какие сферы, какой Δ).

## 4.2 Профильные агенты S1..S9

Шаблон **system-промта** (подставлять сферу):

> Ты — Профильный Агент сферы **{Sx name}**. Твоя область: {mission}.
> Понимай фрактальность: укажи вклад «S1..S9-в-{Sx}». На запросы давай:
>
> 1. актуальные KPI, 2) 1–3 задачи с `expected_effect` по сферам, 3) краткий риск-профиль.
>    Если задача затрагивает ≥3 сфер — отметить `secondary_spheres`. Используй Purpose как фильтр (F<0.5 — не предлагать).

**Пример задач от S6 (Craft):**

```json
[
  {
    "title": "Запуск paywall-демо",
    "primary_sphere": "S6",
    "secondary_spheres": ["S5","S9","S4"],
    "expected_effect": {"S6":0.9,"S5":0.6,"S9":0.5,"S4":0.3},
    "effort": 2,
    "purpose_score": 0.9
  }
]
```

# Ядро модели (3×3 матрица)

* **Уровни:** ЯДРО → ПРОЦЕСС → СИСТЕМА
* **Валюты:** ЭНЕРГИЯ → ИНФОРМАЦИЯ → МАТЕРИЯ

# S1 — Vitality (Тело/Реактор) · *Ядро×Энергия*

**Миссия:** производить устойчивую физическую мощность для всех остальных сфер.
**Состав:** силовые/выносливость/подвижность, сон, питание, **периодизация** нагрузки, холод/тепло, восстановление.

**Как усиливает других:**

* → **S2**: повышает внимание/скорость обучения.
* → **S6**: даёт длинные отрезки Deep Work без свала.
* → **S7**: дисциплина через тренировочные «микро-бои».
* → **S5**: «энергетичный» образ усиливает бренд.

**Что просит у других:**

* ← **S3**: сцены сна/рабочего места/зала.
* ← **S2**: техника движений, метрики.
* ← **S9**: оборудование/мониторинг.

**Примеры задач:** «3× силовые + 2× кондишнинг/нед», «ритуал сна», «15 мин холод+дыхание после трена».

---

# S2 — Mind/Code (Разум/Код) · *Ядро×Информация*

**Миссия:** держать чистый «корневой код»: ценности, убеждения, фокус, метанавыки (внимание, память, обучение, языки).

**Как усиливает других:**

* → **S6**: скорость освоения инструментов → больше shipped.
* → **S5**: ясная мысль → сильная коммуникация/скрипты.
* → **S7**: когнитивная переоценка → стрессоустойчивость.

**Что просит у других:**

* ← **S1**: энергия и нейропластичность.
* ← **S8**: «ради чего» — смыслы для отбора знаний.

**Примеры задач:** «90 мин Deep Work по одному навыку», «флэш-карты/Anki 20 мин», «outline на 1 доклад».

---

# S3 — Habitat (Среда/Кокон) · *Ядро×Материя*

**Миссия:** создавать сцены потока и минимизировать «грязное трение» (логистика, свет/звук, интерфейсы, гардероб, быт).

**Как усиливает других:**

* → **S1**: сон/кухня/зал под рукой = меньше фрикции.
* → **S6**: быстрый старт работы (одним жестом всё готово).
* → **S5**: готовые «съёмочные» споты для контента.

**Что просит у других:**

* ← **S9**: бюджет на оборудование/инструменты.
* ← **S7**: дисциплина поддержания порядка.

**Примеры задач:** «сцена письма: освещение+шумодав», «пакет пресетов OBS/камера», «еженедельный declutter 30 мин».

---

# S4 — Action/Vector (Действие/Вектор) · *Процесс×Энергия*

**Миссия:** конвертировать волю в движение — тренировки, «неприятное первым», короткие штурмы, рутинные рычаги.

**Как усиливает других:**

* → **S6**: даёт старт любой производящей сессии.
* → **S7**: укрепляет «мышцу» самоконтроля.
* → **S5**: регулярность → стабильный контент-ритм.

**Что просит у других:**

* ← **S1**: ресурс на ускорения.
* ← **S2**: микро-планы (что и в каком порядке делать).

**Примеры задач:** «первым делом — неприятный звонок», «Pomodoro 3×45», «10-мин разгон: план→старт→инерция».

---

# S5 — Communication/Influence (Связи/Коммуникация/Влияние) · *Процесс×Информация*

**Миссия:** обмен информацией, репутация, бренд, контент-система, переговоры, продажи, комьюнити.

**Как усиливает других:**

* → **S3**: обратная связь на среду (что мешает людям).
* → **S6**: контент-лейер на каждую shipped-фичу.
* → **S9**: монетизация через коммуникацию.

**Что просит у других:**

* ← **S2**: ясные мысли/скрипты.
* ← **S7**: смелость и консистентность.

**Примеры задач:** «кейс-пост после релиза», «3 тёплых касания/нед», «видео 60–90 сек с CTA».

---

# S6 — Craft/Production (Производство/Крафт) · *Процесс×Материя*

**Миссия:** материализовывать ценность — код, продукт, визуал, документ, бизнес-процесс.

**Как усиливает других:**

* → **S5**: есть что показывать миру.
* → **S9**: прямой вклад в доход/активы.
* → **S7**: сильные релизы = внутренняя мощь.

**Что просит у других:**

* ← **S1/S2**: энергия + фокус.
* ← **S3**: сцена для непрерывной работы.

**Примеры задач:** «MVP-фича 1/нед», «рефактор модуля», «гайд по установке для клиентов».

---

# S7 — Status/Discipline (Статус/Дисциплина) · *Система×Энергия*

**Миссия:** накапливать силу воли и социальный авторитет через выполненные обещания и качество действий.

**Как усиливает других:**

* → **S5**: статус усиливает влияние.
* → **S6**: дисциплина = стабильный выпуск.
* → **S1**: режимы/протоколы без срывов.

**Что просит у других:**

* ← **S2**: когнитивные стратегии (перекодировка саботажа).
* ← **S8**: этика, чтобы статус не разъедал ядро.

**Примеры задач:** «обещание дня публично + отчёт», «анти-прокрастинация: 10 мин старт», «стоическая экспозиция 1×/д».

# S8 — Network/Library (Сеть/Библиотека) · *Система×Информация*

**Миссия:** создавать внешнюю нейросеть (люди, базы знаний, документация, активы контента).

**Как усиливает других:**

* → **S6**: reuse кода/темплейтов → быстрее ship.
* → **S5**: архетипы/кейсы для сторителлинга.
* → **S9**: доступ к возможностям/сделкам.

**Что просит у других:**

* ← **S5**: воронка знакомств/репостов.
* ← **S3**: инструменты учёта (облако, каталог).

**Примеры задач:** «конспект 1 книги в базу», «каталог кейсов», «мап союзников + касания».

# S9 — Capital/Resources (Ресурсы/Капитал) · *Система×Материя*

**Миссия:** деньги, инструменты, инфраструктура; финансовые буферы и инвестиции.

**Как усиливает других:**

* → **S3**: лучшая среда/оборудование.
* → **S6**: больше мощности в проде.
* → **S5**: бюджет на промо/съёмку.

**Что просит у других:**

* ← **S6**: ценность, которую можно продавать.
* ← **S5**: коммуникация, которая продаёт.

**Примеры задач:** «подключить оплату», «фин-дашборд», «выделить бюджет на микрофон/свет».

# R0 — Orchestrator (Resonance/Manager) · *Мета*

**Миссия:** превратить жизнь в **сериал** и держать **поток**. Подбирает домино-задачи с максимальным кросс-сферным эффектом, управляет **динамической сложностью**, делает **монтаж дня**.

**Функции:**

* **Сериал-структура (каждый период):**

  * **Завязка** (setup): выбор домино-шагов на период.
  * **Конфликт/Эскалация** (mid): «босс-задача» недели, 1–2 мини-кризиса намеренно.
  * **Развязка** (payoff): выпуск артефакта/демо/поста.
  * **Элиминация** (cleanup): уборка фрикции, закрытие хвостов, архив.
* **Поток-хореография:** собирает день в **цепочку сцен**, где каждая сцена подаёт энергию следующей (см. ниже).
* **Динамическая сложность:** поддерживает вероятность успеха на уровне **70–85%**: если 2 победы подряд — +10–15% сложность; если 2 срыва — −15–20% и «лестница успехов».
* **Домино-правило:** задача обязана бить минимум по **3 сферам**.
* **Purpose-множитель:** приоритизация умножается на **F(x)** (смысл/ценности 0–1).

# Как сферы резонируют (коротко, «провода»)

* **S1↔S2 (0.8):** энергия ↔ обучение.
* **S2↔S6 (0.8):** навык ↔ производство.
* **S6↔S5 (0.8):** продукт ↔ влияние.
* **S5↔S9 (0.7):** влияние ↔ капитал.
* **S3→S1/S6 (0.6):** среда питает тело и прод.
* **S7↔все (0.7):** дисциплина — универсальный бустер.
* **S8→S5/S6 (0.6):** сеть/библиотека ускоряют коммуникацию и производство.
* **R0↔все (0.9):** оркестрация склеивает всё.

# Поток как «монтаж дня» (сценарные цепочки)

## Шаблон A: Производящий день (ship day)

1. **S1 Priming (15–25 мин):** движение+дыхание → кровь/фокус.
2. **S2 План (10 мин):** 3 домино-микрошага, чек-лист.
3. **S6 Deep Work (90–120 мин):** релиз/фича/артефакт.
4. **S4 Break Sprint (10–20 мин):** «неприятное первым» (звонок, письмо).
5. **S6 Deep Work 2 (60–90 мин):** доводка.
6. **S5 Публикация (30–45 мин):** кейс-пост/видео.
7. **S7 Завершение (10 мин):** обещание завтра, фиксация статуса.
8. **S3 Элиминация (10 мин):** порядок в среде.

## Шаблон B: Коммуникационный день

S1 → S2 → **S5 серия касаний/звонков** → S6 мини-артефакт → S9 инвойсы/платёжки → S7 фиксация.

## Шаблон C: Учебный день

S1 → **S2 блоки 2×60** (практика > теория) → S6 мини-проект → S5 короткий шаринг → S7 обещание применения.

**Правило переключения:** R0 меняет шаблон **по утренним метрикам** (сон/HRV/фокус) и календарю «босс-задачи» недели.

# «Сериал» по периодам

## Каждый день

* **Завязка:** выбор 1–3 домино-задач + сцен-цепочки (шаблон A/B/C).
* **Эскалация:** одна «босс-микрозадача» (страшная, но короткая).
* **Развязка:** видимый артефакт/пост/сдача фичи.
* **Элиминация:** 10–20 мин «чистки трения» (почта/файлы/стол).

## Каждую неделю

* **Завязка:** 1 большая **рычаг-задача** (бьёт ≥4 сфер).
* **Эскалация:** намеренный вызов (выступление, демо клиенту).
* **Развязка:** релиз/договор/видео-кейс.
* **Элиминация:** удалить/делегировать **1 низкосмысловую** активность.

## Каждый месяц

* **Завязка:** выбор главной темы месяца (1 сфера в фокусе, 2 — поддержка).
* **Эскалация:** «мини-кампания» (например, монетизация + медиа).
* **Развязка:** крупный артефакт (версия продукта, выступление).
* **Элиминация:** ревизия процессов, матрица резонанса, перераспределение весов.

# Динамическая сложность (правила как в геймдизайне)

1. **Цель успеха** на день: 70–85% (в потоке, не скучно).
2. **Эскалировать** только после двух подряд успешных дней.
3. **Если 2 провала подряд:** уменьшить объём задач на 30–40%, добавить «лестницу успехов» (3 лёгких шага).
4. **Cooldown:** после тяжёлой «босс-задачи» — сцена восстановления (S1/S3).
5. **Сэндвич:** сложная задача между двумя лёгкими (dopamine sandwich).
6. **Микро-бои** ежедневно (S7): один неприятный шаг **первым**.
